

## Plan: Точное соответствие мобильной версии блога референсу + навигация

Сравнивая референс с текущим кодом, нужны следующие изменения:

### Файл: `src/pages/Blog.tsx`

**1. Категория-badge в карточках статей — заполненный вместо outlined**
На референсе badge категории имеет цветной фон (как `bg-primary/10 text-primary`), а не просто `border`. Заменить outlined span на заполненный pill:
```tsx
// Было: border border-border text-muted-foreground
// Стало: bg-primary/10 text-primary (или привязать цвет к категории)
```

**2. Миниатюра у первой статьи**
На референсе первая карточка имеет маленькое изображение справа (~64x64, rounded). Добавить thumbnail для первых N статей из существующих изображений в `/public/images/work/`. Layout карточки: `flex justify-between`, текст слева, изображение справа (только если есть).

**3. Кнопка «Назад к статьям» в BlogPost.tsx**
Добавить явную кнопку возврата в `src/pages/BlogPost.tsx` — рядом с breadcrumbs или под заголовком. Это улучшит навигацию на мобильном.

**4. Категории — обновить emoji для точного соответствия**
На референсе: Законы = ⚖️, Кейсы = 💼. В текущем коде: Законодательство = 📜, Случаи из практики = 📋. Но категории в `types.ts` уже "Законы" и "Кейсы", так что emoji в `categoryIcons` нужно обновить:
- "Законы" → ⚖️
- "Кейсы" → 💼

### Файлы для изменения:
- `src/pages/Blog.tsx` — badge стиль, thumbnail, emoji
- `src/pages/BlogPost.tsx` — кнопка возврата к списку
