

# Исправление Hero: убрать ротатор, вернуть один фон

## Проблема
На главную страницу (Hero) был ошибочно добавлен ротатор из 5 фоновых изображений. Задача была — использовать загруженные фото для посадочных страниц районов и округов, а не для главной.

## Что будет сделано

### 1. Hero.tsx — убрать ротатор, вернуть статичный фон
- Удалить массив `HERO_BACKGROUNDS` (5 картинок)
- Заменить на одну константу `HERO_BACKGROUND = '/images/backgrounds/moscow-panorama-sunset.jpg'`
- Удалить state-переменные `currentBgIndex`, `nextBgIndex`, `isTransitioning`
- Удалить useEffect для prefetch и ротации (2 useEffect)
- Убрать импорт `useState`
- Упростить JSX: один div с фоном вместо двух с crossfade-логикой
- Карточки с фото (fast-response, certificates, guarantee) остаются без изменений

### Результат
- Главная страница: один стабильный фон (панорама Москвы)
- Страницы районов (`neighborhoodImages.ts`): 16 районов с уникальными hero-фото и галереями — без изменений, уже корректно
- Страницы округов (`districtImages.ts`): 9 округов с уникальными фото — без изменений, уже корректно
