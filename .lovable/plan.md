

# Сплит llm-optimized-articles.ts на тематические модули

## Проблема

Файл `llm-optimized-articles.ts` уже содержит 1965 строк (15 статей). При продолжении работы (~70 батчей по 3 статьи) он вырастет до 13000+ строк, что приведет к обрезанию при редактировании.

## Решение

Разбить LLM-оптимизированные статьи на тематические файлы внутри новой папки `src/data/blog/llm/`, сохранив единый экспорт.

## Структура файлов

```text
src/data/blog/llm/
  pests.ts          -- клопы, тараканы, грызуны, сезонность (9001, 9003, 9004, 9007, 9008, 9013)
  methods.ts        -- методы обработки, подготовка, озонирование (9002, 9005, 9010, 9011, 9012)
  legal-commercial.ts -- офис, документы, кейсы (9006, 9009, 9014, 9015)
  index.ts          -- реэкспорт объединенного массива
```

## Принцип распределения будущих статей

| Кластер | Файл | Содержание | Макс. статей |
|---------|------|------------|-------------|
| Вредители | `pests.ts` | Клопы, тараканы, муравьи, крысы, блохи, комары, моль, клещи, сезонность | ~40 |
| Методы и препараты | `methods.ts` | Туман, озон, аэрозоли, подготовка, профилактика, выбор службы | ~30 |
| Юридическое и коммерция | `legal-commercial.ts` | Роспотребнадзор, документы, офисы, рестораны, гостиницы, арендаторы | ~25 |
| Советы и безопасность | `safety-tips.ts` | Дети, животные, после болезни, FAQ-сборники (создается по мере роста) | ~20 |

При необходимости каждый файл можно дробить дальше (например, `pests-insects.ts` и `pests-rodents.ts`).

## Изменения в index.ts

Файл `src/data/blog/index.ts` будет импортировать из `./llm/` вместо `./llm-optimized-articles`:

```text
- import { llmOptimizedArticles } from './llm-optimized-articles';
+ import { llmOptimizedArticles } from './llm';
```

## Порядок работы

1. Создать `src/data/blog/llm/pests.ts` -- перенести 6 статей про вредителей
2. Создать `src/data/blog/llm/methods.ts` -- перенести 5 статей про методы
3. Создать `src/data/blog/llm/legal-commercial.ts` -- перенести 4 статьи про юридику и кейсы
4. Создать `src/data/blog/llm/index.ts` -- объединить и экспортировать массив
5. Обновить импорт в `src/data/blog/index.ts`
6. Удалить старый `src/data/blog/llm-optimized-articles.ts`

## Что не меняется

- Интерфейс `BlogArticle`, типы, компоненты -- без изменений
- Система приоритетов в `index.ts` -- без изменений (просто другой путь импорта)
- Все slug, id, контент статей -- переносятся как есть

## Технические детали

- Каждый тематический файл экспортирует `BlogArticle[]`
- `src/data/blog/llm/index.ts` объединяет все массивы через spread и экспортирует как `llmOptimizedArticles`
- Будущие батчи добавляются в соответствующий тематический файл (не в один гигантский)
- Максимальный размер одного файла -- ~40 статей (~5000 строк), после чего дробим дальше
