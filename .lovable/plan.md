

# Переработка калькулятора для мобильной версии

## Проблема 1: Калькулятор слишком длинный на мобильных

Клиент открывает калькулятор, вводит данные, видит цену -- но телефон и кнопка "Позвоните мне" за пределами экрана. Нужно скролить, чтобы понять что делать дальше.

## Проблема 2: На страницах услуг кнопка "Рассчитать стоимость" перекидывает на главную

В `ServicePage.tsx` и `ServiceSubpage.tsx` используется `navigate('/#calculator')` вместо открытия модала.

---

## Изменения

### Файл 1: `src/components/Calculator.tsx`

При `isModal` скрыть следующие блоки:

| Что | Строки | Действие |
|---|---|---|
| "Заполнить полную форму заявки" | 632-639 | Скрыть при `isModal` |
| Аккордеон "Расширенные настройки" | 654-741 | Скрыть при `isModal` |
| "Сравнить методы обработки" | ~744+ | Скрыть при `isModal` |
| "Показать график цены от площади" | ~800+ | Скрыть при `isModal` |

Оставить при `isModal`:
- Тип помещения (6 кнопок)
- Площадь (слайдер + инпут)
- Блок результата (цена, скидка, итог)
- QuickCallForm (телефон + "Позвоните мне")
- Кнопка "Оставить заявку" (CompactRequestButton)
- Подпись о персональных данных

Уменьшить отступы: `space-y-4` вместо `space-y-6` при `isModal`.

### Файл 2: `src/pages/ServicePage.tsx`

- Добавить `useState` для `showCalculator` и импорт `CalculatorModal`
- Заменить `navigate('/#calculator')` на `setShowCalculator(true)`
- Добавить `<CalculatorModal>` в JSX

### Файл 3: `src/pages/ServiceSubpage.tsx`

- Аналогично: состояние + модал вместо навигации на главную

## Результат

- Все важные элементы калькулятора видны на мобильных без скролла
- Калькулятор открывается на текущей странице услуг, а не перекидывает на главную
