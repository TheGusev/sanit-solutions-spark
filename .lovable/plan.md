

# Исправление мобильной версии блога — полная ширина + удобные категории

## Проблемы (видно на скриншоте)
1. Контент в "контейнере" — `px-4` + `container max-w-6xl` сжимают содержимое на мобильных
2. Категории в горизонтальном скролле — неудобно, не видно что есть ещё
3. Карточки статей не на полную ширину экрана

## Изменения в `src/pages/Blog.tsx`

| Элемент | Было | Станет |
|---|---|---|
| Breadcrumbs секция | `px-4`, container max-w-6xl | `px-2 md:px-4`, убрать лишний отступ |
| Hero секция | `px-4`, container max-w-6xl | `px-3 md:px-4` |
| Категории | `overflow-x-auto` горизонтальный скролл | `flex-wrap` на мобильных тоже, кнопки меньше (`text-[11px]`, `px-2.5`), wrap всегда |
| Сетка статей | `px-4`, `gap-6` | `px-2 md:px-4`, `gap-4 md:gap-8`, одна колонка на мобильных уже без отступов |
| CTA секция | `px-4` | `px-3 md:px-4` |

Ключевое изменение категорий: вместо `overflow-x-auto` на мобильных — `flex-wrap` всегда, кнопки компактнее чтобы помещались в 2-3 ряда. Убрать `shrink-0` с кнопок.
